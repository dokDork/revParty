Lo script python è chiamato reverseParty.py.
Lo script python si troverà in una cartella chiamata reverseParty. Sotto reverseParty ci saranno le seguenti cartelle:
- engine che conterrà le seguenti sottocartelle:
-- powershell-cmd: cartella che conterrà il file powershell-cmd.txt con la lista di tutti i comandi powershell e il file powershell-cmd-noParametri.txt che contiene comandi powershell che solitamente non hanno parametri.
-- second-stage: cartella che conterrà un certo numero di file. Ogni file rappresenta una reverse shell con il codice in chiaro scaricata da uno stager come secondo stage
-- stager: cartella che contiene un certo numero di file. Ogni file rappresenta un possibile stager con il codice in chiaro.
-- out: cartella contenente i risultati delle varie elaborazioni
-- stuff: cartella che contiene alcuni file a supporto della creazione della backdoor e del trojan. I file contenuti sono: il file LAUNCHERNAME
-- listener: cartella che contiene i listener da lanciare in funzione della

Lo script dovrà presentare nelle prime righe alcune variabili che chi usa lo script potrà modificare facilmente. Le variabili andranno quindi commentate nello script con una descrizione e un esempio dei valori che possono assumere.
Le variabili sono le seguenti:

- LHOST: ip pubblico su cui inviare la reverse shell
- LPORT: porta su cui ci si aspetta di ricevere la reverse shell (es. 9001)
- ATTACKER-URL: URL da cui verranno scaricati il second-stage, lo zip della backdoor e l'iso della (es. http://192.168.1.11 oppure http://myDomain.com/stuff)

- SECONDNAME: è il nome che avrà il second stage (es. second.txt)
- STAGERNAME: è il nome dello stager (es. stager.txt)
- LAUNCHERNAME: è il nome del file con buona reputazione per windows che consente di bypassare smartscreen (es. launcher.bat)
- EXENAME: è il nome dello stager/trojan reso eseguibile. Deve essere eseguito dal launcher (es. ''ps2pdf'.exe)
- ZIPNAME: è il nome dello zip che contiene launcher e stager in formato exe (es. postscript.zip)
- ISONAME: è il nome dell'iso che contiene il launcher e trojan in formato zip

- WIN-IP: IP della macchina windows da usare per trasformare i file ps1 in exe (es. 192.168.1.10)
- WIN-USER: utente per accedere alla macchina windows (es. user)
- WIN-PASS: password per accedere alla macchina windows (es. myPassword)

- TROJAN-FE: è il nome del file di front end (es. update_k897867.msu oppure image.jpg) che viene visualizzato dopo che la backdoor viene installata
- ICONNAME: è il nome del file.ICO che viene utilizzato per rendere piu credibile il trojan.EXE
- TROJANNAME: è il nome.PS1 che descrive il comportamento del trojan (scaricando la backdoor e installandola e poi attivando il file TROJAN-FE)
- TROJAN-URL: è l'URL da cui scaricare il trojan (es. https://raw.githubusercontent.com/dokDork/dokDork.github.io/main/soloemapuoaccedere)

- [ATTACKER-IP]: va sostituito con il parametro LHOST
- [ATTACKER-PORT]: va sostituito con il parametro LPORT

- a questo punto dovranno essere offuscati i comandi di powershell che di solito non hanno parametri.
